error setting certificate verify locations

发表于 2019-10-03 | 热度： ℃

hexo博客上传错误

今个像往常般上传hexo博客文件时，突然报错，问题有二

一、 git 检查换行格式报错

问题如图

原因 ：

git 在 Windows下默认是CRLF作为换行符，git add 提交时，检查文本中有LF 换行符（linux系统里面的），则会告警。所以问题的解决很简单，让git忽略该检查即可

解决方法

git config --global core.autocrlf false

二 ca-bundle.crt 文件路径不对

Error: fatal: unable to access https://github.com/xxx/xxx.github.io.git: error setting certificate verify locations:
CAfile: D:/Programming Tools/Git/mingw64/ssl/certs/ca-bundle.crt
CApath: none

问题如图

原因：

前段时间修改了D盘里的Programming Tools文件夹的名字，将其空格去掉，即ProgrammingTools，因为Git也安装在该文件夹里，故导致Git的 ca-bundle.crt 文件路径发生了错误

解决办法：

git config --system http.sslacinfo "D:/Programming Tools/Git/mingw64/ssl/certs/ca-bundle.crt

其中”D:/xxx”需换成自己ca-bundle.crt的路径

一定要用反斜杠/，不能用\
